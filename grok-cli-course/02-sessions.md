# Урок 2 — Сесії

## Про що цей урок

Сесія Grok CLI — це збережена розмова з контекстом, планом і історією tool calls. Навчимося створювати, відновлювати, розгалужувати та ділитися сесіями.

**Головна мета — не вивчити всі команди Grok CLI, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.**

---

## Навіщо керувати сесіями

- **Нова задача** → `/new`, щоб не тягнути старий контекст
- **Продовження вчорашньої роботи** → `/resume` або `/sessions`
- **Експеримент без ризику** → `/fork` від поточної точки
- **Організація** → `/rename` для зрозумілої назви в списку

---

## Команди

### `/new`

| | |
|---|---|
| **Що робить** | Починає нову сесію; контекст розмови скидається |
| **Коли** | Інша задача, інший проєкт, «чистий лист» |
| **Приклад** | `/new` |
| **Результат** | Нова сесія; порожня історія |
| **Помилки** | — |
| **Ризик** | **Низький** (але втратиш незбережений контекст поточної сесії — стара сесія залишається в `/sessions`) |

---

### `/resume`

| | |
|---|---|
| **Що робить** | Відновлює попередню сесію зі списку |
| **Коли** | Продовжити роботу після `/quit` або перезапуску `grok` |
| **Приклад** | `/resume` |
| **Результат** | Picker з минулими сесіями |
| **Помилки** | Немає сесій — лише `/new` |
| **Ризик** | **Низький** |

---

### `/sessions`

| | |
|---|---|
| **Що робить** | Список усіх минулих сесій для вибору |
| **Коли** | Шукаєш конкретну розмову за назвою або датою |
| **Приклад** | `/sessions` |
| **Результат** | Список: назва, час, активна сесія позначена |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/fork`

| | |
|---|---|
| **Що робить** | Копіює поточну сесію в нову гілку розмови |
| **Коли** | Спробувати інший підхід, не втрачаючи основний тред |
| **Приклад** | `/fork` |
| **Результат** | Нова сесія з тією ж історією до точки fork |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/rename <title>`

| | |
|---|---|
| **Що робить** | Перейменовує поточну сесію |
| **Коли** | Після `/new` — дати змістовну назву (`fix-auth`, `refactor-api`) |
| **Приклад** | `/rename fix-auth` |
| **Результат** | Назва оновлена в `/sessions` |
| **Помилки** | Без аргументу — може запросити назву в UI |
| **Ризик** | **Низький** |

---

### `/share`

| | |
|---|---|
| **Що робить** | Ділиться сесією через URL (якщо увімкнено в акаунті) |
| **Коли** | Показати колезі хід розв'язання; не для секретів |
| **Приклад** | `/share` |
| **Результат** | URL для перегляду сесії |
| **Помилки** | Sharing вимкнено в акаунті — помилка або недоступно |
| **Ризик** | **Середній** — у сесії можуть бути фрагменти коду, ключі, внутрішні шляхи |

---

### `/session-info`

| | |
|---|---|
| **Що робить** | Метадані: id, назва, модель, режим, час |
| **Коли** | Перевірити, в якій сесії ти зараз; яка модель активна |
| **Приклад** | `/session-info` |
| **Результат** | Title, Model, Mode (plan / default) |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/home`

| | |
|---|---|
| **Що робить** | Повертає на welcome screen TUI |
| **Коли** | Хочеш вибрати: нова сесія, resume, без `/quit` |
| **Приклад** | `/home` |
| **Результат** | Екран вітання; сесія не видаляється |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/quit` (alias `/exit`)

| | |
|---|---|
| **Що робить** | Вихід з Grok CLI TUI |
| **Коли** | Закінчив роботу; перед цим перевір `git status` |
| **Приклад** | `/quit` |
| **Результат** | Повернення в shell; сесія збережена для `/resume` |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

## Сценарій: два підходи до однієї задачі

```
/rename refactor-api
# ... довга розмова ...
/fork
/rename refactor-api-v2-graphql
# експериментуєш з іншим API
```

Якщо v2 не вдалась — повертаєшся до оригіналу через `/sessions`.

---

## Практичне завдання

1. Запусти `grok`. `/new` → `/rename test-session-01`.
2. `/session-info` — запиши назву та модель.
3. `/fork` → `/rename test-session-01-fork`.
4. `/sessions` — переконайся, що обидві сесії в списку.
5. `/home` → `/resume` → обери `test-session-01`.
6. `/quit`. Знову `grok` → `/resume` — сесія на місці.
7. **Тренажер:** модуль «2. Сесії» в [тренажері](../index.html).

---

## Міні-чекліст

- [ ] `/new` для нової задачі, `/resume` для продовження
- [ ] `/rename` одразу після старту — інакше в `/sessions` хаос
- [ ] `/fork` перед ризиковим експериментом
- [ ] `/share` тільки без секретів у транскрипті
- [ ] Перед `/quit` — `git status` у shell
