# Урок 6 — Extensions

## Про що цей урок

Hooks, plugins, skills і MCP — розширення Grok CLI. Команди `/hooks`, `/plugins`, `/skills`, `/mcps` відкривають **один** extensions modal на різних вкладках.

**Головна мета — не вивчити всі команди Grok CLI, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.**

---

## Що таке extensions

| Вкладка | Призначення |
|---------|-------------|
| **Hooks** | Lifecycle hooks — дії до/після tool calls, сесій |
| **Plugins** | Додаткові можливості, інтеграції |
| **Skills** | User-invocable навички — можуть стати slash-командами (`/commit`) |
| **MCP** | Model Context Protocol — зовнішні tools (filesystem, GitHub, DB…) |

Усі чотири команди → той самий modal, різна вкладка за замовчуванням.

---

## Команди

### `/hooks`

| | |
|---|---|
| **Що робить** | Extensions modal → вкладка **Hooks** |
| **Коли** | Перевірити/налаштувати lifecycle hooks |
| **Приклад** | `/hooks` |
| **Результат** | Список hooks, trust/disable |
| **Помилки** | Новий hook — може потребувати підтвердження trust |
| **Ризик** | **Середній** — hook може виконувати shell-код автоматично |

---

### `/plugins`

| | |
|---|---|
| **Що робить** | Extensions modal → вкладка **Plugins** |
| **Коли** | Встановити або вимкнути plugin; подивитися tools |
| **Приклад** | `/plugins` |
| **Результат** | Installed / discoverable plugins |
| **Помилки** | Plugin несумісний з версією CLI |
| **Ризик** | **Середній** — plugins розширюють surface area агента |

---

### `/skills`

| | |
|---|---|
| **Що робить** | Extensions modal → вкладка **Skills** |
| **Коли** | Подивитися user-invocable skills; знайти `/commit` тощо |
| **Приклад** | `/skills` |
| **Результат** | Список skills: `commit`, `review`… |
| **Помилки** | Skill не встановлено — slash-команда не з'явиться |
| **Ризик** | **Середній** — skill може запускати git/shell workflow |

---

### `/mcps`

| | |
|---|---|
| **Що робить** | Extensions modal → вкладка **MCP** |
| **Коли** | Підключити GitHub, DB, browser; перевірити доступні tools |
| **Приклад** | `/mcps` |
| **Результат** | MCP servers: filesystem, github… |
| **Помилки** | Сервер не стартує — перевір config і PATH |
| **Ризик** | **Середній–високий** — MCP дає агенту доступ до зовнішніх систем |

---

## Сценарій: аудит перед роботою в новому репо

```
/mcps      # які зовнішні tools доступні?
/skills    # які slash-workflows можу викликати?
/hooks     # чи є auto-run скрипти?
/plugins   # що встановлено глобально?
```

Якщо щось невідоме — вимкни або не давай trust до розуміння, що робить extension.

---

## MCP vs Skills

| | MCP | Skills |
|---|-----|--------|
| **Виклик** | Агент через tools | Ти через `/skill-name` або агент |
| **Приклад** | GitHub API, postgres | `/commit`, `/review` |
| **Ризик** | Доступ до мережі/БД | Shell/git операції |

---

## Практичне завдання

1. `grok` → `/hooks` — що показує modal?
2. `/plugins` → `/skills` → `/mcps` — переконайся, що це той самий modal, різні вкладки.
3. У `/skills` знайди user-invocable skills (якщо є). Запиши їхні імена.
4. `/btw що таке MCP?` — коротке пояснення без виходу з modal-контексту задачі.
5. **Тренажер:** модуль «6. Extensions».

---

## Міні-чекліст

- [ ] `/hooks`, `/plugins`, `/skills`, `/mcps` — один modal, чотири вкладки
- [ ] Перед trust на hook/plugin — читаю, що він виконує
- [ ] MCP — мінімально необхідні сервери
- [ ] Skills дивлюся в `/skills` перед викликом `/commit`
- [ ] Після змін extensions — `/new` або `/fork` для чистого контексту
