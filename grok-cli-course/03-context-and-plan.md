# Урок 3 — Контекст і план

## Про що цей урок

Context window — обмежений ресурс. Plan mode — планування без зайвих правок. Команди `/context`, `/compact`, `/rewind` і `/plan` тримають сесію під контролем.

**Головна мета — не вивчити всі команди Grok CLI, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.**

---

## Режими сесії

У TUI **`Shift+Tab`** перемикає session modes, зокрема **Plan** і **Always-approve**.

**Plan mode:** write-tools заблоковані, крім файлу session plan. Grok спочатку планує, може поставити уточнююче питання — потім ти схвалюєш реалізацію.

---

## Команди

### `/context`

| | |
|---|---|
| **Що робить** | Показує, скільки context window зайнято |
| **Коли** | Перед великою задачею; після довгої розмови; перед вставкою великих логів |
| **Приклад** | `/context` |
| **Результат** | `Context: 42% used` (приблизно) |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/plan`

| | |
|---|---|
| **Що робить** | Вмикає plan mode і показує поточний session plan |
| **Коли** | Складна зміна в коді; рефакторинг; невпевненість у підході |
| **Приклад** | `/plan` |
| **Результат** | Plan mode ON; write-tools заблоковані (крім plan file) |
| **Помилки** | — |
| **Ризик** | **Низький** — навпаки знижує ризик передчасних правок |

---

### `/compact` [context]

| | |
|---|---|
| **Що робить** | Стискає історію розмови, звільняє context window |
| **Коли** | `/context` показує >60–70%; після завершення етапу роботи |
| **Приклад** | `/compact` або `/compact останні зміни в auth` |
| **Результат** | Summarized history; менший % context |
| **Помилки** | Занадто рано — втратиш деталі, які ще потрібні |
| **Ризик** | **Середній** — деталі старих повідомлень можуть зникнути з активного контексту |

---

### `/compact-mode`

| | |
|---|---|
| **Що робить** | Перемикає щільніший UI layout у TUI |
| **Коли** | Мало місця на екрані; багато виводу в терміналі |
| **Приклад** | `/compact-mode` |
| **Результат** | Denser layout ON/OFF |
| **Помилки** | Не плутати з `/compact` — це лише вигляд, не контекст |
| **Ризик** | **Низький** |

---

### `/rewind`

| | |
|---|---|
| **Що робить** | Відкочує розмову до попередньої точки (checkpoint) |
| **Коли** | Агент пішов не туди; хочеш «скасувати» гілку діалогу |
| **Приклад** | `/rewind` |
| **Результат** | Вибір checkpoint; історія скорочується |
| **Помилки** | Файлові зміни на диску **не** відкочуються автоматично — потрібен `git checkout` |
| **Ризик** | **Середній** — плутанина, якщо забути відкотити git |

---

### `/btw <question>`

| | |
|---|---|
| **Що робить** | Side-question — питання «по дорозі» без зламу основного флоу |
| **Коли** | Швидке уточнення терміну, API, синтаксису під час задачі |
| **Приклад** | `/btw як працює MCP?` |
| **Результат** | Коротка відповідь; основна задача продовжується |
| **Помилки** | — |
| **Ризик** | **Низький** |

---

### `/usage`

| | |
|---|---|
| **Що робить** | Token і credit usage для поточної сесії |
| **Коли** | Довга сесія; багато tool calls; контроль витрат |
| **Приклад** | `/usage` |
| **Результат** | In/out tokens, credits (залежить від акаунту) |
| **Помилки** | — |
| **Ризик** | **Низький** |

## Сценарій: велика задача без переповнення контексту

```
/context
/plan
# опиши задачу: «Міграція auth на JWT — тільки план»
# після схвалення плану — Shift+Tab вийти з plan або дозволити правки

# ... багато кроків ...
/context
/compact завершені кроки міграції
/btw що таке refresh token rotation?
```

Якщо агент змінив файли не так — `/rewind` + `git diff` + `git checkout -- <file>`.

---

## Практичне завдання

1. `grok` → `/context` — запиши початковий %.
2. `/plan` — постав задачу «Склади план тестів для модуля X, не чіпай код».
3. `/btw що таке context window?` — переконайся, що основний тред не зламався.
4. `/usage` — подивись tokens.
5. `/compact-mode` — перемкни UI; `/compact` — знову `/context`, порівняй %.
6. **Тренажер:** модуль «3. Контекст і план».

---

## Міні-чекліст

- [ ] Перед великою задачею — `/context` і `/plan`
- [ ] `/compact` після етапу, не в середині складного дебагу
- [ ] `/rewind` ≠ git revert — файли перевіряю через `git diff`
- [ ] `/btw` для дрібних питань, не для нової великої задачі
- [ ] `Shift+Tab` — знаю, як увійти/вийти з plan mode
