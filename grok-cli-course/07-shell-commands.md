# Урок 7 — Shell-команди

## Про що цей урок

Команди від `xai-grok-shell`: пам'ять розмови, persistent memory, офлайн-consolidation і генерація медіа. Це не TUI-core, а shell-шар поверх сесії.

**Головна мета — не вивчити всі команди Grok CLI, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.**

---

## Core TUI vs shell-provided

| Джерело | Приклади |
|---------|----------|
| TUI (pager-local) | `/new`, `/context`, `/plan` |
| `xai-grok-shell` | `/flush`, `/memory`, `/dream`, `/imagine` |
| Skills | `/commit`, `/local:commit` |

Shell-команди працюють у тому ж `grok>` prompt, але логіка — в shell-шарі.

---

## Команди

### `/flush`

| | |
|---|---|
| **Що робить** | Примусово записує conversation memory на диск |
| **Коли** | Перед `/quit` після важливої розмови; перед перезавантаженням |
| **Приклад** | `/flush` |
| **Результат** | `✓ Conversation memory flushed to disk` |
| **Помилки** | Диск повний / permissions — рідко |
| **Ризик** | **Низький** |

---

### `/memory`

| | |
|---|---|
| **Що робить** | Пошук і редагування persistent memory entries |
| **Коли** | Агент «пам'ятає» зайве; хочеш додати правило про стиль проєкту |
| **Приклад** | `/memory` |
| **Результат** | Memory editor — search & edit |
| **Помилки** | — |
| **Ризик** | **Середній** — неправильний memory впливає на **майбутні** сесії |

---

### `/dream`

| | |
|---|---|
| **Що робить** | Offline memory-consolidation pass |
| **Коли** | Після довгої серії сесій; «упорядкувати» пам'ять у фоні |
| **Приклад** | `/dream` |
| **Результат** | Consolidation started (offline) |
| **Помилки** | Може тривати довго — не блокує TUI повністю |
| **Ризик** | **Середній** — змінює, як агент згадує минуле |

---

### `/imagine <prompt>`

| | |
|---|---|
| **Що робить** | Генерує зображення з текстового prompt |
| **Коли** | UI mockup, іконка, ілюстрація — не для коду |
| **Приклад** | `/imagine minimal app icon dark theme` |
| **Результат** | Image generation queued |
| **Помилки** | Ліміти акаунту / content policy |
| **Ризик** | **Низький** для коду (не чіпає репозиторій); витрати credits |

---

### `/imagine-video <prompt>`

| | |
|---|---|
| **Що робить** | Генерує відео з текстового prompt |
| **Коли** | Короткий demo-ролик, motion mockup |
| **Приклад** | `/imagine-video loading spinner animation` |
| **Результат** | Video generation queued |
| **Помилки** | Довша черга; вищі credits |
| **Ризик** | **Низький** для файлів проєкту; **середній** за вартістю |

---

## Сценарій: пам'ять після тижня роботи

```
/memory          # перевір, що збережено глобально
# видали застарілі entries про старий API
/flush           # збережи поточну розмову
/dream           # consolidation у фоні
/quit
```

Для коду й рефакторингу — `/compact` і git, не `/dream` як заміна.

---

## Практичне завдання

1. `grok` → `/memory` — переглянь entries (якщо є).
2. `/flush` — після короткої розмови.
3. `/imagine simple terminal icon` — лише якщо є credits; інакше прочитай опис у [тренажері](../index.html).
4. Порівняй: `/compact` (контекст сесії) vs `/memory` (між сесіями).
5. **Тренажер:** модуль «7. Shell commands».
