# Урок 4 — Модель і UI

## Про що цей урок

Вибір моделі, тема TUI, багаторядковий ввід і feedback — налаштування, які впливають на зручність і якість відповідей, але не змінюють файли напряму.

**Головна мета — не вивчити всі команди Grok CLI, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.**

---

## Коли міняти модель

| Задача | Підхід |
|--------|--------|
| Швидкий огляд коду, дрібні правки | Швидша / компактніша модель |
| Архітектура, складний рефакторинг | Потужніша модель |
| Довга сесія | `/usage` після перемикання — різні моделі = різна вартість |

Перевірка після зміни: `/session-info` або `/model` без аргументів.

---

## Команди

### `/model` [name]

| | |
|---|---|
| **Що робить** | Перемикає активну модель Grok |
| **Коли** | Перед складною задачею; коли відповіді занадто поверхневі або повільні |
| **Приклад** | `/model grok-3` |
| **Результат** | `● grok-3 active` |
| **Помилки** | Невідома назва моделі — перевір каталог у picker |
| **Ризик** | **Низький** (вищі витрати tokens/credits на потужніших моделях) |

---

### `/theme` [name]

| | |
|---|---|
| **Що робить** | Змінює кольорову тему TUI |
| **Коли** | Читабельність у темній/світлій кімнаті; демо на проекторі |
| **Приклад** | `/theme dark` |
| **Результат** | Theme: dark |
| **Помилки** | Невідома тема — список у picker |
| **Ризик** | **Низький** |

---

### `/multiline`

| | |
|---|---|
| **Що робить** | Toggle багаторядкового вводу в composer |
| **Коли** | Довгі промпти, списки вимог, вставка diff або JSON |
| **Приклад** | `/multiline` |
| **Результат** | `Multiline: ON` — Enter додає рядок, не відправляє |
| **Помилки** | Забув вимкнути — плутанина з відправкою повідомлення |
| **Ризик** | **Низький** |

---

### `/feedback` [text]

| | |
|---|---|
| **Що робить** | Надсилає feedback по поточній сесії в xAI |
| **Коли** | Баг, дивна поведінка tool, пропозиція по UX |
| **Приклад** | `/feedback compact втратив важливий контекст` |
| **Результат** | `✓ Feedback sent` |
| **Помилки** | Без мережі — помилка відправки |
| **Ризик** | **Низький** (не включай секрети та API keys у текст) |

---

## Сценарій: довгий промпт для code review

```
/multiline
# вставляєш чекліст review:
# - security
# - tests
# - naming
/model grok-3
/theme dark
```

Після відправки — `/multiline` знову, щоб повернути однорядковий режим.

---

## Практичне завдання

1. `grok` → `/session-info` — яка модель за замовчуванням?
2. `/model` — перемкни модель (якщо доступно кілька). `/session-info` — підтверди зміну.
3. `/theme` — спробуй іншу тему.
4. `/multiline` ON → напиши промпт з 3 рядків → відправ → `/multiline` OFF.
5. `/feedback тестовий відгук з курсу` (опційно).
6. **Тренажер:** модуль «4. Модель і UI».

---

## Міні-чекліст

- [ ] `/model` перед складною задачею, не посеред tool run
- [ ] `/usage` після довгої сесії на «важкій» моделі
- [ ] `/multiline` для довгих інструкцій
- [ ] `/feedback` без секретів у тексті
- [ ] `/theme` — особиста зручність, не впливає на код
